# Skill: community-template

Добавление пользовательского шаблона устройства в репозиторий wb-community.

## Использование

```
/community-template
```

## Шаги выполнения

### Шаг 0. Запросить шаблон у пользователя

Перед началом работы запросить JSON шаблона устройства в любом удобном виде:
- Путь к файлу на диске или URL (GitHub, Gist, прямая ссылка и т.д.)
- Или попросить скопировать содержимое JSON прямо в чат

Не переходить к Шагу 1, пока шаблон не получен.

### Шаг 1. Проверить и положить шаблон в репозиторий

**Файл:** `templates/wb-mqtt-serial/config-<бренд>-<модель>.json`
- Имя файла: lowercase, слова через дефис
- Папка: всегда `templates/wb-mqtt-serial/`, если не указано другое

**Первый коммит — исходник:**
1. Создать ветку: `community/<бренд>-<модель>`
2. Положить файл шаблона как есть, без изменений: `templates/wb-mqtt-serial/config-<бренд>-<модель>.json`
3. Сделать коммит с сообщением: `Add original <бренд>-<модель> template`

После этого — проверить шаблон по чеклисту и внести правки отдельным коммитом.

**Чеклист проверки шаблона:**

#### Структура
- [ ] Есть обязательные поля верхнего уровня: `device_type`, `title`, `group`, `device`
- [ ] В `device` есть: `name`, `id`, `channels`, `translations`
- [ ] `groups` определены, если каналов > 3-4
- [ ] `title` использует ключ перевода (например `tmk_n_template_title`), а не прямой текст

#### Именование MQTT-топиков (name каналов)
- [ ] `snake_case`, все слова с маленькой буквы
- [ ] Не более 4 слов и цифр
- [ ] Без знаков препинания, скобок, спецсимволов (%$#&)
- [ ] Без единиц измерения в имени (для этого есть `units`)

#### Переводы (translations)
- [ ] Есть секция `en` и `ru`
- [ ] EN: Each Word Capitalized (Ram Used, Board Temperature)
- [ ] RU: Только первое слово с большой (Температура платы, Использованная память)
- [ ] Исключение — аббревиатуры: VOC, MQTT, RAM всегда капсом
- [ ] Переведены: title шаблона, названия групп, названия каналов, enum_titles

#### Тексты и оформление
- [ ] Тексты понятны непрограммисту
- [ ] `е` вместо `ё`
- [ ] Программистские кавычки `"`, не типографские
- [ ] Дефис `-`, не тире `—`
- [ ] Десятичный разделитель — точка: `20.5`
- [ ] Комментарии к параметрам: короткие (1-3 предложения), без точки в конце
- [ ] Нет капслока (кроме аббревиатур)
- [ ] Единицы измерения только в `units`, не в названиях каналов

#### Минимализм
- [ ] Каждый канал и параметр нужен — лишние удалены
- [ ] Если без элемента можно обойтись — его нет
- [ ] Названия понятны без комментариев
- [ ] Удалены регистры/каналы для изменения адреса Modbus (slave id) и скорости (baudrate) — эти параметры настраиваются через веб-интерфейс, не через шаблон

### Создание PR

1. Внести правки по чеклисту и сделать второй коммит
2. Создать PR в `wirenboard/wb-community`
3. Отправить пользователю ссылку на PR

Не ждать merge — сразу параллельно переходить к Шагу 2 и Шагу 3.

Не приступать к созданию страницы, пока логин и пароль не получены.

#### Создание скриншотов на контроллере

Для вики-страницы нужны два скриншота: настройки устройства в конфигураторе и виджет на странице Devices.

**Подготовка:**
1. Спросить у пользователя IP контроллера, логин/пароль (по умолчанию `root`/`wirenboard`)
2. Проверить доступность контроллера (ping + SSH через `sshpass`)
3. Переключить язык веб-интерфейса на английский: Settings → Web UI → UI locale → English
4. Загрузить шаблон на контроллер по SCP в `/etc/wb-mqtt-serial.conf.d/templates/`
5. Перезапустить **оба** сервиса: `wb-mqtt-serial` и `wb-mqtt-confed` (оба нужны, чтобы шаблон появился в веб-интерфейсе)

**Добавление устройства:**
6. Открыть `http://<IP>/#!/serial-config` (Device manager)
7. Нажать "Manually add any device", найти шаблон по имени в комбобоксе "Device type", выбрать, нажать "Add"
8. Установить slave id = 1

**Скриншот 1 — настройки устройства (settings):**
9. Установить viewport 1280×900
10. Сделать скриншот страницы serial-config с выбранным табом устройства
11. Обрезать: убрать навигационную панель слева и пустое пространство снизу

**Настройка таймаута:**
12. Открыть диалог "Parameters", включить "Response timeout (ms)", закрыть диалог
13. Установить response timeout = 60000 мс (чтобы каналы не стали красными до снятия скриншота)
14. Нажать "Save config"

**Скриншот 2 — виджет устройства (widget):**
15. Перейти на `http://<IP>/#!/devices`
16. Найти карточку устройства (`.card.card-primary` с именем устройства в `h4.card-title`)
17. Сделать скриншот карточки — каналы **не должны** быть красными
18. Обрезать с небольшим отступом (~2px)

**Очистка:**
19. Удалить устройство из конфигурации через веб-интерфейс (кнопка Delete на табе устройства, затем Save config)
20. Удалить файл шаблона с контроллера
21. Перезапустить оба сервиса

Отправить пользователю ссылку на созданную страницу вики.

### Шаг 3. Добавить в таблицу поддерживаемых устройств

Таблица находится на странице: https://wiki.wirenboard.com/wiki/Wbincludes:Unsupported_Devices

**Формат добавления:**
- Найти подходящую категорию в таблице (Датчики климата, Счётчики электроэнергии и т.д.)
- Добавить внутреннюю вики-ссылку на созданную страницу: `[[Имя_страницы|Отображаемое название]]`
- Если в категории уже есть устройства — добавить через разделитель ` • `
- Если подходящей категории нет — создать новую строку в таблице

**Пример:** добавление устройства «ТМК-Н» в категорию «Счетчики тепла»:
```mediawiki
| Счетчики тепла
| [[VIS.T2-TC| ВИС.Т2-ТС]] • [[TMK-N | Тепловычислитель ТМК-Н]]
```

Отправить пользователю ссылку на таблицу.

### Проверка перед публикацией

Шаги 1–3 выполняются параллельно. Перед переходом к Шагу 4 убедиться, что всё принято:

- [ ] PR влит в `main`
- [ ] Страница устройства в вики утверждена (апрувнута)
- [ ] Правка в таблице поддерживаемых устройств утверждена

Попросить пользователя подтвердить каждый из этих пунктов. Не переходить к Шагу 4, пока все три не выполнены.

### Шаг 4. Сделать новость в Телеграм

Сформировать пост для Телеграм-канала.

Отдать пользователю текст поста + картинки (фото устройства, скриншоты виджета и настроек). Попросить проверить и опубликовать в канал: https://t.me/+cBtJCikRYNQyZDgy

**Формат поста:**

```
Добавили пользовательский шаблон <тип устройства> <Название устройства>

<эмодзи> <Название устройства> — это <краткое описание: что это, для чего, ключевые характеристики>. Подробнее в документации (<ссылка на вики-страницу>).

💬 Особая благодарность нашему пользователю за создание шаблона! Мы добавили его в наш репозиторий (<https://github.com/wirenboard/wb-community>).

Если у вас есть собственные наработки, делитесь ими! Чтобы сделать процесс проще, мы подготовили рекомендации по созданию шаблонов (<https://wiki.wirenboard.com/wiki/Connecting_Third_Party_Devices_to_Wiren_Board>).
```

**К посту прикрепить:** фото устройства из интернета + скриншоты (виджет и настройки).

**Если устройств несколько** — описать каждое отдельным абзацем со своим эмодзи, в конце один общий блок с благодарностью.

**Правила текста (из руководства по стилю):**
- Обращаемся на «вы» с маленькой буквы
- Текст для пользователей, не для разработчиков — описываем что устройство делает, а не как устроено
- Компактно и читаемо, без лишних слов
- Эмодзи перед абзацем, пробел между эмодзи и текстом. Аккуратно с красными эмодзи
- Кавычки-ёлочки «слово» в русском тексте
- Тире длинное (—), не дефис
- Десятичный разделитель — точка
- Единицы измерения по-русски с пробелом: 400 см, 100 Гц
- Интервалы: от 4 до 400 см или 4…400 см
- Названия технологий как в оригинале: RS-485 Modbus RTU (без запятой)
- Название компании — Wiren Board (два слова через пробел)
- Без орфографических и пунктуационных ошибок

### Шаг 5. (опционально) Создать идею интеграции

Спросить у пользователя: «Устройство интересное? Стоит создать идею полноценной интеграции в Проекты Wiren Board?»

Если пользователь говорит да:
- Сформировать заголовок тикета: «Интеграция <Название устройства>»
- Краткое описание: что за устройство, для чего применяется, какой протокол
- Оценка: неделя
- Отдать пользователю для создания тикета в Проектах Wiren Board

## Примеры правильного оформления

### Имена каналов
| Неправильно | Правильно |
|---|---|
| `Status of Compressor 1` | `compressor_1_status` |
| `Version of the Application` | `application_version` |
| `Ram Used` | `ram_used` |

### Переводы
| EN (каждое слово с большой) | RU (только первое) |
|---|---|
| Supply Temperature | Температура подачи |
| Fan Speed | Скорость вентилятора |
| Accumulated Energy | Накопленная энергия |
